Fluxo de Autenticação
Registro em duas etapas com verificação por código, JWT com access + refresh tokens e silent refresh automático no cliente.

Fluxo de Registro em 2 Etapas

sequenceDiagram participant U as Usuário participant FE as Frontend participant BE as Backend participant DB as SQLite participant EM as EmailService U->>FE: Preenche formulário (nome, email, senha, role) FE->>BE: POST /auth/register/initiate BE->>BE: Valida Zod DTO BE->>BE: Gera código 6 dígitos + bcrypt(senha) BE->>DB: Cria PendingRegistration (TTL 5min) BE->>EM: sendVerificationCode(email, código) BE-->>FE: 200 OK {message} FE-->>U: "Verifique seu e-mail" U->>FE: Digita código recebido FE->>BE: POST /auth/register/confirm {email, code} BE->>DB: Busca PendingRegistration (verifica expiração) alt Código válido BE->>DB: Cria User + Seller (se role=SELLER) BE->>DB: Deleta PendingRegistration BE-->>FE: 201 Created {accessToken, refreshToken} FE-->>U: Login automático else Código inválido/expirado BE-->>FE: 400 Bad Request FE-->>U: Erro — reenviar código end

Fluxo de Login e Silent Refresh

sequenceDiagram participant U as Usuário participant FE as Frontend participant HC as HttpClient participant BE as Backend U->>FE: POST /auth/login {email, senha} FE->>BE: authLimiter (15 req/15min) BE->>BE: bcrypt.compare(senha, hash) alt Credenciais válidas BE-->>FE: {accessToken, refreshToken, user} FE->>FE: localStorage.setItem(neon_access_token) FE->>FE: localStorage.setItem(neon_refresh_token) FE-->>U: Dashboard else Inválidas BE-->>FE: 401 Unauthorized end Note over HC,BE: Requests subsequentes com token expirado HC->>BE: GET /orders (Authorization: Bearer ) BE-->>HC: 401 Unauthorized HC->>HC: Enfileira request pendente HC->>BE: POST /auth/refresh {refreshToken} alt Refresh válido BE-->>HC: {accessToken novo} HC->>HC: Salva novo token HC->>BE: GET /orders (novo token) ← retry BE-->>HC: 200 OK else Refresh expirado HC->>FE: Dispara auth:session-expired FE->>FE: AuthContext.logout() FE-->>U: Redireciona /login end
